Glossary

AI 통계 계산 예시 전문 용어 설명

FDA, 식약처, CE 관점의 AI 통계 예시를 읽을 때 자주 나오는 규제, 검증, 샘플 수 계획 용어를 한국어로 정리했습니다.

Benchmark

AI가 비교해야 하는 기준 성능입니다. 기존 제품, 전문가 판독, 문헌 수치, 내부 목표값 등이 benchmark가 될 수 있습니다.

Regulatory Lens

같은 기능이라도 FDA, 식약처, CE가 무엇을 중요하게 보는지 정리한 시각입니다. 예시 탭에서는 이 관점을 문장과 점검 포인트로 보여줍니다.

Acceptance Criteria

시험 성공 여부를 판단하는 사전 기준입니다. 보통 primary endpoint의 신뢰구간 하한이 목표 기준을 넘는지로 정의합니다.

Validation Dataset Independence

최종 검증 데이터가 학습 또는 튜닝 데이터와 겹치지 않아야 한다는 원칙입니다. 겹치면 실제보다 성능이 좋아 보일 수 있습니다.

External Validation

개발에 쓰지 않은 외부 기관 또는 다른 시기의 데이터로 성능을 검증하는 것입니다. 일반화 가능성을 보여줄 때 매우 중요합니다.

Design Effect

site, reader, lesion, frame처럼 데이터가 서로 독립적이지 않을 때 필요한 표본 수를 더 늘려주는 보정 계수입니다.

Protected Subgroup

성별, 연령, 장비, 중증도처럼 별도로 성능을 확인해야 하는 하위 집단입니다. 규제 검토에서 자주 강조되는 항목입니다.

Reference Review Failure Rate

정답 판정이 실패하거나 불확실해져 최종 분석에서 제외될 가능성을 의미합니다. 실제 모집 수를 더 늘려야 할 근거가 됩니다.

Locked Model

검증 이후 모델이 바뀌지 않는다고 가정하는 운영 방식입니다. 변경이 생기면 별도 검증이나 새로운 제출 전략이 필요할 수 있습니다.

PCCP

Predetermined Change Control Plan의 약자입니다. AI가 이후 어떤 범위 안에서 업데이트될 수 있는지와 검증 방법을 미리 정하는 계획입니다.

Human Factors Validation

사용자가 AI 표시를 실제 환경에서 올바르게 이해하고 사용하는지 검증하는 절차입니다. 실시간 AI에서는 특히 중요합니다.

Real-world Data Bridge

실사용 환경 데이터를 이용해 기존 검증 결과와 실제 운영 환경 사이를 연결하는 접근입니다. 데이터 적합성과 편향 관리가 핵심입니다.
